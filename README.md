# Grant Application Skills

A practical skill set for helping people write stronger funding applications, including grants, fellowships, scholarships, prizes, accelerators, and concept notes.

This repository focuses on one core outcome: helping applicants produce clear, credible, criteria-aligned applications without inventing facts or losing their authentic voice.

## What this repository contains

- A primary skill definition in skills/SKILL.md
- Supporting reference modules for story, proposal structure, budgets, funder fit, and templates
- A packaged skill file in skills/grant-application.skill

## Repository structure

- skills/SKILL.md: Main guidance and end-to-end workflow for application support
- skills/storytelling-and-persuasion.md: Narrative frameworks and ethical persuasion techniques
- skills/proposal-anatomy.md: Standard proposal sections and what reviewers look for
- skills/budgets-and-financials.md: Budget construction, cost categories, and financial credibility checks
- skills/funder-fit-and-criteria-check.md: Funder research, fit checks, and rubric mapping
- skills/templates.md: Reusable templates for pitches, concept notes, cover letters, and reporting
- skills/grant-application.skill: Packaged artifact for compatible skill runtimes

## Core workflow in this skill

1. Gather full context about the call and the applicant.
2. Check funder fit before investing heavy drafting time.
3. Build the story and logic spine first.
4. Draft directly to criteria and section limits.
5. Build or validate a realistic budget.
6. Run a final criteria-check pass before submission.

## Design principles

- Accuracy first: no fabricated numbers, partners, quotes, or outcomes
- Criteria alignment over generic prose
- Authentic applicant voice over over-polished language
- Safety and consent in storytelling
- Honest fit decisions when opportunities are weak matches

## How to use

Use this repository as:

- A source skill for AI assistant workflows focused on funding applications
- A reference library for proposal coaching and review
- A template bank for faster drafting and iteration

Typical flow:

1. Start with skills/SKILL.md.
2. Pull in the relevant support file(s) based on the section being drafted.
3. Validate every draft against the call requirements and scoring criteria.
4. Use templates in skills/templates.md for fast, consistent outputs.

## Contributing

Contributions are welcome.

Suggested contribution pattern:

1. Keep guidance practical and field-tested.
2. Preserve the non-negotiables (no fabrication, no forced fit, consent and safety).
3. Prefer concrete examples and clear checklists over abstract advice.
4. Update related files when changing workflow assumptions.
